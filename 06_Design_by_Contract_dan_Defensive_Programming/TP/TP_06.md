## Soal

Diberikan dua kode yang sama-sama melakukan operasi pembagian. Pertama menggunakan asersi, kedua menggunakan eksepsi.

const assert = require('assert');

function divide(a, b) {
  assert(typeof a === 'number' && typeof b === 'number', 'Nilai harus bilangan bulat');

  assert(b !== 0, 'Tidak bisa pembagian dengan nol');

  return a / b;
}

function divide(a, b) {
  if (typeof a !== "number" || typeof b !== "number") {
    throw new TypeError("Nilai harus bilangan bulat");
  }

  if (b === 0) {
    throw new Error("Tidak bisa pembagian dengan nol");
  }

  return a / b;
}

try {
  const result = divide(10, 2);
  console.log("Hasilnya adalah:", result);
} catch (error) {
  console.error("Error:", error);
}

## JAWABAN

Kapan kita  menggunakan asersi atau eksepsi?

Gunakan Asersi jika:
- Kita ingin memeriksa kondisi yang "seharusnya tidak mungkin terjadi" jika kode Kita benar.
- Memeriksa private method yang hanya dipanggil oleh modul internal Kita sendiri.
- Mengecek invariant (kondisi yang harus selalu benar) di tengah-tengah algoritma yang kompleks.

Gunakan Eksepsi jika:
- Kita memvalidasi data yang datang dari luar (input pengguna, API, database).
- Fungsi tersebut adalah bagian dari Public API atau library yang akan digunakan oleh orang lain.
- Kondisi kesalahan tersebut adalah sesuatu yang bisa dipulihkan (misalnya meminta pengguna memasukkan angka   
  kembali jika mereka memasukkan nol).

penggunaan eksepsi biasanya jauh lebih tepat untuk lingkungan produksi. Hal ini dikarenakan pembagian dengan nol atau salah tipe data adalah kesalahan input yang sangat umum terjadi di tingkat pengguna. Jika kita hanya menggunakan asersi, aplikasi akan berhenti berfungsi setiap kali pengguna melakukan kesalahan kecil, yang tentu saja akan merusak pengalaman pengguna.

Idealnya, asersi digunakan selama fase pengembangan untuk menangkap logika yang cacat dengan cepat, sedangkan eksepsi diimplementasikan untuk membangun sistem yang tahan banting (robust) dan mampu menangani ketidakpastian dunia nyata. Strategi terbaik bukanlah "sepenuhnya asersi" atau "sepenuhnya eksepsi", melainkan menerapkan "Design by Contract". Jadi gunakan asersi untuk kontrak internal antar fungsi, dan gunakan eksepsi untuk melindungi perbatasan aplikasi dari data luar yang tidak valid
